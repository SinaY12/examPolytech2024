Docker est un logiciel permettant la gestion de conteneurs, ceux-ci permettant d'enfermer une application (son code, ses dépendances et toutes ses configuration dans un environnement isolé.
Docker est beaucoup utilisé dans le domaine du développement car il permet d'exécuter une application dans n'importe quel environnement car tout ce dont a besoin l'application se trouve dans le conteneur, chaque conteneur fonctionne indépendamment, ce qui permet d'éviter que des applications de deux conteneurs différents rentrent en conflit. Il permet donc souvent de simplifier le déploiement d'application et permet une meilleure gestion des ressources.

17.
Le but de l'intégration continue (CI) est de pouvoir automatiser le processus de fusion de code dans une branche partagée entre plusieurs développeurs et d'effectuer automatiquement les tests pour que le projet fonctionne.
Le déploiement continu (CD) permet de faire en sorte de tout le temps préparer le code pour qu'il soit prêt à être déployer à tout moment.
La CI/CD est utile pour de nombreux points, elle permet un déploiement plus rapide, avec moins d'erreurs. Elle permet aussi une meilleure collaboration entre collègues, et améliore donc la productivité.
